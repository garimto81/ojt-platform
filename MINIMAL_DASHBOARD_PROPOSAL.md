## 📊 현재 문제점 분석

### 1. 정보 과다 (Information Overload)

**Dashboard 메인 페이지**:
- ❌ 4개 통계 카드 (완료 일수, 진행률, 포인트, 순위)
- ❌ 현재 모듈 카드 (레슨 목록 4개)
- ❌ 리더보드 TOP 5
- ❌ 바로가기 2개
- ❌ 헤더 버튼 2개 (학습 시작, 로그아웃)

**Learning 페이지**:
- ❌ 7개 Day 카드 (각각 3-4개 레슨)
- ❌ Day별 학습 목표
- ❌ 상태 표시 (잠김, 진행중, 완료)
- ❌ 포인트, 시간, 레슨 수 등 메타 정보

**Layout (사이드바)**:
- ❌ 6개 메인 메뉴
- ❌ 2개 Admin 메뉴
- ❌ 상단 바에 포인트, 알림, 아바타

```
신규 직원의 질문: "지금 뭘 해야 하지?"
현재 대시보드의 답변: "너무 많은 선택지..."
```

- **의사결정 피로**: 너무 많은 옵션
- **집중력 분산**: 순위, 포인트 등 부차적 정보

## 🎯 추천: Progressive Minimal 모드

**이유**:
1. ✅ **즉시 사용 가능**: 고민 없이 "다음 레슨" 클릭
2. ✅ **점진적 탐색**: 필요시 "전체보기"로 확장
3. ✅ **기존 코드 재활용**: 큰 구조 변경 없이 레이아웃만 변경
4. ✅ **심리적 부담 최소**: 한 번에 하나만 보여줌

### 구현 우선순위

**Phase 1: Quick Win (1-2시간)**
```
1. Dashboard 메인 페이지 단순화
   - 통계 카드 4개 → 진행률 바 1개
   - 현재 모듈 카드 → "다음 레슨" 버튼 1개
   - 리더보드, 바로가기 → 제거
```

**Phase 2: Layout 최적화 (2-3시간)**
```
2. 사이드바 축소
   - 메뉴 6개 → 햄버거 메뉴로 숨김
   - 상단 바 → 로고 + 진행률 + 프로필만
```

**Phase 3: Learning 페이지 간소화 (3-4시간)**
```
3. Learning 페이지
   - 7개 Day 카드 전체 표시 → 현재 Day만 펼침
   - Day별 학습 목표 → 접기
   - 레슨 아이콘 축소
```

## 💬 FAQ

**Q: 기존 기능들(리더보드, 포인트)은 어떻게 되나요?**
A: 숨겨두고 "전체 보기" 또는 프로필 페이지에서 확인 가능합니다.

**Q: Admin은 어떻게 접근하나요?**
A: 햄버거 메뉴(☰)에서 Admin 섹션 접근 가능합니다.

**Q: 모바일에서는?**
A: Progressive Minimal 디자인은 모바일 퍼스트로 설계되어 더 좋은 경험을 제공합니다.

**Q: 기존 코드를 많이 바꿔야 하나요?**
A: 아니요. 새 페이지를 만들고 라우팅만 변경하면 됩니다.
